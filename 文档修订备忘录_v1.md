## 修订 #1：虚拟形象 V1 方案（PRD 09 ↔ PRD 11 矛盾解决）

### 矛盾描述

| 文档 | 位置 | 原文 |
|------|------|------|
| PRD 11《个性化与人格层》§4.1 | 核心决策（已定板） | "V1 路线 A：静态 icon（圆形头像 + 5 种基础表情）" |
| PRD 09《HMI 多屏协同》§5.6 H-F6 | 功能点展开 | "不做 V1 静态 icon（推翻 5/2 的判断 → 动态形象是 V1 体验差异化的核心）" |

### 解决决策（以本备忘录为准）

**V1 采用动态 Live2D 形象（采纳 PRD 09 §5.6 H-F6 判断，废弃 PRD 11 §4.1 旧决策）。**

理由：
1. PRD 09 §5.6 是更晚的文档，且有充分论述（竞品研究、成本分析、技术路径）
2. 沈砚架构回答 ⑥-12 已确认 Live2D 技术可行（<4 TOPS，<200MB，成本可控）
3. 静态 icon 对标竞品（蔚来 NOMI / 理想同学 / 极氪 Eva）差距明显，差异化不足

### 更新内容

**PRD 11《个性化与人格层》需更新的段落（待下次文档修订时执行）：**
- §4.1 V1 静态 icon → 改为：~~V1 路线 A：静态 icon~~ **（已废弃，见 PRD 09 §5.6 H-F6 和修订备忘录 #1）**
- §4.2 V1.5 动态形象 → 改为：~~V1.5 路线~~ **（V1 已上动态，V1.5 升级 3D）**

### V1 虚拟形象最终定义（以本条为准）

| 项目 | V1 决策 |
|------|---------|
| 形态 | Live2D 2D 骨骼动画（不做静态 icon，不做 3D 写实） |
| 状态数 | 6 状态：休眠/唤醒/聆听/思考/成功/失败 |
| 驾驶中 | 简化为表情+光效，停止大幅动画 |
| 换肤 | V1 预置 5-10 套官方模板，不支持用户自上传 |
| 位置 | 中控顶部状态条（不占主屏） |
| 算力 | <4 TOPS（GPU 分担），<200MB 内存 |
| V1.5 | 升级 3D 形象（预研从 V1 发布后立即开始） |
