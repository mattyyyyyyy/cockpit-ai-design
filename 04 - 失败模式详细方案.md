**04 - 失败模式详细方案**

**失败模式详细方案**

**文档版本**: v0.1**创建时间**: 2026-05-10**作者**: 沈砚(后端架构师 + Agent 编排)**目标**: 给出座舱 AI 的失败模式清单,每个失败模式配检测方法 + 兜底方案 + 代码级示例

**1. 单点失败**

**1.1 云端大模型不可用**

**失败场景**: 云端大模型服务挂了、限流、超时

**检测方法**:

|  |
| --- |
| Python async def call\_cloud\_model(prompt, timeout=3.0):  try:  response = await cloud\_api.call(prompt, timeout=timeout)  return response  except TimeoutError:  logger.error("cloud\_model\_timeout")  raise  except APIError as e:  if e.status\_code == 429: # 限流  logger.error("cloud\_model\_rate\_limited")  elif e.status\_code >= 500: # 服务端错误  logger.error("cloud\_model\_unavailable")  raise |

**兜底方案**: 切端侧小模型

|  |
| --- |
| Python async def call\_model\_with\_fallback(prompt):  try:  return await call\_cloud\_model(prompt, timeout=3.0)  except (TimeoutError, APIError):  logger.info("fallback\_to\_local\_model")  return await call\_local\_model(prompt) |

**监控指标**:

* cloud\_model\_timeout\_rate: 云端超时率
* fallback\_to\_local\_rate: 降级到端侧的比例

**告警阈值**: 降级率 >10% 持续 5 分钟

**1.2 ASR 服务挂了**

**失败场景**: 云端 ASR 服务不可用

**检测方法**:

|  |
| --- |
| Python async def call\_asr(audio\_data, timeout=2.0):  try:  text = await asr\_api.recognize(audio\_data, timeout=timeout)  return text  except TimeoutError:  logger.error("asr\_timeout")  raise  except APIError:  logger.error("asr\_unavailable")  raise |

**兜底方案**: 端侧 ASR

|  |
| --- |
| Python async def recognize\_with\_fallback(audio\_data):  try:  return await call\_asr(audio\_data, timeout=2.0)  except (TimeoutError, APIError):  logger.info("fallback\_to\_local\_asr")  return await local\_asr.recognize(audio\_data) |

**监控指标**:

* asr\_timeout\_rate: ASR 超时率
* asr\_accuracy\_cloud: 云端 ASR 准确率
* asr\_accuracy\_local: 端侧 ASR 准确率

**告警阈值**: 云端 ASR 超时率 >5% 持续 3 分钟

**1.3 某个 Agent 超时**

**失败场景**: Agent 调用超时(如导航 Agent 查 POI 慢)

**兜底方案**: 降级到规则引擎或跳过该 Agent

**监控指标**:

* agent\_timeout\_rate: 各 Agent 超时率
* agent\_fallback\_rate: 降级到规则引擎的比例

**告警阈值**: 某个 Agent 超时率 >10% 持续 5 分钟

**1.4 CAN 总线故障**

**失败场景**: 车控指令发送失败(硬件故障)

**兜底方案**: 提示用户手动操作

**监控指标**: can\_error\_rate(CAN 总线错误率)

**告警阈值**: CAN 错误率 >0.1% 立刻告警(硬件故障)

**1.5 网络完全断开**

**失败场景**: 隧道、地库、偏远地区网络不可用

**兜底方案**: 切 L3 安全兜底(端侧状态机,只保留车控 + 离线导航)

**监控指标**: network\_unavailable\_rate / offline\_mode\_usage

**告警阈值**: 网络不可用率 >30% 持续 10 分钟(可能是区域性故障)

**2. 级联失败**

**2.1 意图识别错误 → 参数补全错误 → 执行错误**

**失败场景**: 串行 Agent 链路,上游错误传染到下游

**兜底方案**: 低置信度时拒绝执行,多轮确认

|  |
| --- |
| Python async def execute\_with\_confirmation(text):  intent, intent\_conf = await intent\_recognition(text)  if intent\_conf < 0.8:  return {"status": "need\_clarification", "message": "您是想做什么?"}  slots, slots\_conf = await slot\_filling(intent, text)  if slots\_conf < 0.8:  return {"status": "need\_confirmation", "message": f"您是想{intent},对吗?"}  return await execute(intent, slots) |

**监控指标**: low\_confidence\_intent\_rate / clarification\_rate

**告警阈值**: 澄清率 >20% 持续 10 分钟(模型质量下降)

**2.2 某个 Agent 慢 → 聚合等待 → 整体超时**

**失败场景**: 并行 Agent 中某个慢,聚合等待导致整体超时

**兜底方案**: 超时后放弃该 Agent,用部分结果

|  |
| --- |
| Python async def parallel\_agents\_with\_timeout(input\_data):  tasks = {"agent1": agent1.call(input\_data), "agent2": agent2.call(input\_data)}  results = {}  for name, task in tasks.items():  try:  results[name] = await asyncio.wait\_for(task, timeout=3.0)  except asyncio.TimeoutError:  logger.warning(f"agent\_timeout: {name}")  results[name] = None  return aggregate\_partial\_results(results) |

**监控指标**: parallel\_agent\_timeout\_rate / partial\_result\_rate

**2.3 工具调用失败 → Agent 重试 → 雪崩**

**失败场景**: 工具调用失败,Agent 无限重试,打垮下游服务

**兜底方案**: 限流 + 熔断(CircuitBreaker)

|  |
| --- |
| Python class CircuitBreaker:  def \_\_init\_\_(self, failure\_threshold=5, timeout=60):  self.failure\_count = 0  self.failure\_threshold = failure\_threshold  self.timeout = timeout  self.state = "closed" # closed/open/half\_open    async def call(self, func, \*args):  if self.state == "open":  if time.time() - self.last\_failure\_time > self.timeout:  self.state = "half\_open"  else:  raise CircuitBreakerOpenError()  try:  result = await func(\*args)  if self.state == "half\_open":  self.state = "closed"  self.failure\_count = 0  return result  except Exception:  self.failure\_count += 1  self.last\_failure\_time = time.time()  if self.failure\_count >= self.failure\_threshold:  self.state = "open"  raise |

**告警阈值**: 熔断器打开 → 立刻告警

**3. 上下文一致性失败**

**3.1 多 Agent 看到不同版本的上下文**

**失败场景**: 异步更新导致上下文不一致

**兜底方案**: 版本号 + 锁,版本不一致时重试

|  |
| --- |
| Python class Context:  def \_\_init\_\_(self):  self.data = {}  self.version = 0  self.lock = asyncio.Lock()    async def update(self, key, value):  async with self.lock:  self.data[key] = value  self.version += 1 |

**告警阈值**: 版本不一致率 >5% 持续 5 分钟

**3.2 用户中途改变意图,Agent 还在执行旧任务**

**失败场景**: 用户说"导航到公司",Agent 正在规划路线,用户又说"算了,去机场"

**兜底方案**: 新意图到达时立刻取消旧任务(TaskManager.cancel\_task)

|  |
| --- |
| Python class TaskManager:  async def start\_task(self, task\_id, task\_func):  if self.current\_task\_id:  await self.cancel\_task(self.current\_task\_id)  self.current\_task\_id = task\_id  self.tasks[task\_id] = asyncio.create\_task(task\_func()) |

**告警阈值**: 取消率 >30% 持续 10 分钟(用户体验差)

**4. 资源耗尽**

**4.1 并发请求过多 → 云端限流**

**兜底方案**: 限流后降级到端侧或排队

|  |
| --- |
| Python class RateLimiter:  def \_\_init\_\_(self, max\_qps=100):  self.max\_qps = max\_qps  self.tokens = max\_qps  self.last\_update = time.time()  self.lock = asyncio.Lock()    async def acquire(self):  async with self.lock:  now = time.time()  self.tokens = min(self.max\_qps, self.tokens + (now - self.last\_update) \* self.max\_qps)  self.last\_update = now  if self.tokens >= 1:  self.tokens -= 1  return True  return False |

**告警阈值**: 限流次数 >100/分钟

**4.2 端侧内存不足 → 模型加载失败**

**兜底方案**: 卸载低优先级模型

**告警阈值**: 内存使用率 >90% 持续 5 分钟

**4.3 存储满 → 日志/缓存写入失败**

**兜底方案**: 清理旧数据或停止写入

**告警阈值**: 磁盘使用率 >95% 立刻告警

**5. 安全失败**

**5.1 车控误触发**

**失败场景**: 语音误识别导致错误车控(如误开车窗)

**兜底方案**: 高风险操作需用户确认

|  |
| --- |
| Python def is\_high\_risk\_command(command):  return command in ["open\_window", "unlock\_door", "open\_trunk"]  async def execute\_vehicle\_control(command, params):  if is\_high\_risk\_command(command):  return {"status": "need\_confirmation", "message": f"确认要{command}吗?"}  return await execute(command, params) |

**告警阈值**: 高风险指令率突增 >50%(可能是误识别)

**5.2 紧急打断失败**

**失败场景**: 用户说"停",但系统没有立刻中断

**兜底方案**: 紧急关键词立刻中断所有操作

|  |
| --- |
| Python EMERGENCY\_KEYWORDS = ["停", "stop", "取消", "cancel"]  async def handle\_user\_input(user\_input):  if any(k in user\_input.lower() for k in EMERGENCY\_KEYWORDS):  await cancel\_all\_tasks()  return {"status": "interrupted"}  return await process\_intent(user\_input) |

**告警阈值**: 打断延迟 >500ms(安全风险)
