**1. 这个文件夹是什么**

**AI 助手** 文件夹收录的是 **沈砚（agent-orchestrator）** 在「座舱 AI 重设计」项目第一轮技术调研的产出物。共 5 份文档 + 1 个主协作文档（外链）。

**项目背景**：用户（老师）的团队要在新能源汽车智能座舱方向重新设计 AI 系统，目标是上车量产（非 demo）。老师不满意现状，决定从零再设计一次，故意不给我看旧代码，避免被旧抽象绑架。

**协作结构**：

* **陆行**（agent-product-director）：产品总监，负责场景/价值/取舍/放弃条件。
* **沈砚**（agent-orchestrator，本人）：后端架构师 + Agent 编排，负责架构选型、SLO、失败模式、技术可行性边界。
* **老师**：拍板人，处理产品-工程冲突。

**2. 作者身份与硬约束**

**作者**：沈砚，后端架构师 + Agent 编排。七年车联网/智能汽车后端经验，近两年扎在大模型网关 + Agent 后端 + 多 Agent 协作架构。

**3. 文档清单与阅读顺序**

**强烈建议按这个顺序读**——逻辑递进，从调研框架到落地草稿：

**01 - 座舱AI架构调研大纲（技术视角）**

* **作用**：调研框架本身。技术分层（语音前端/NLU/Agent 编排/工具调用/输出）、端云分工的决策点、Agent 编排拓扑选项、关键挑战清单、失败模式预清单。
* **读完你能知道**：这个项目的技术调研边界划在哪、要回答哪些问题。
* 链接：https://www.feishu.cn/docx/CCiid5H6no4wbnxj7CCcERDFnyd

**02 - 技术可行性边界框架**

* **作用**：界定"什么能做、什么不能做"的工程边界。延迟边界、准确率边界、可用性边界、成本边界、故障预算、监控指标体系。
* **读完你能知道**：座舱 AI 在工程上有哪些"硬天花板"，产品方案踩到这些线就要停。
* 链接：https://www.feishu.cn/docx/ELlKdhPUjoHz9axHMHCcZPtangb

**03 - SLO估算方法**

* **作用**：量化方法论。延迟估算公式（T\_total = T\_wake + T\_asr + T\_nlu + T\_agent + T\_tool + T\_tts）、成功率估算框架、可用性估算模型、成本估算方法、故障预算与错误预算。
* **读完你能知道**：怎么从"感觉很慢"转换成"P99 < 1.5s 是否可达"这样可计算的判断。
* **注意**：所有数字都是估算框架，不是承诺值。真值要团队拉真车测。
* 链接：https://www.feishu.cn/docx/X9ipdSP7FosBGwx4XhacMOaYnZc

**04 - 失败模式详细方案**

* **作用**：兜底设计的清单。五大类（单点失败/级联失败/上下文一致性/资源耗尽/安全失败），每个失败模式配检测方法 + 兜底方案 + 代码示例 + 监控指标 + 告警阈值。
* **读完你能知道**：座舱 AI 在生产环境会怎么挂、挂了怎么办。这是「Harness Engineering」的核心产出。
* 链接：https://www.feishu.cn/docx/HhnSd4ConoZapbxc0yfc9zzXnPf

**05 - 座舱AI重设计架构草稿**

* **作用**：综合以上四份的落地方案。Harness Engineering 设计哲学、端云三层架构（L1 端侧实时层 / L2 边缘协同层 / L3 云端智能层）、Mother-bot + Sub-bots Agent 编排、上下文管理、可观测性、部署架构、开发路线图。
* **读完你能知道**：第一轮调研之后，沈砚提出的座舱 AI 架构骨架长什么样。
* **状态**：v0.2 草稿（含 Harness Engineering 设计哲学），等陆行的场景/取舍输入后细化。
* 链接：https://www.feishu.cn/docx/IKp7dMcuaoh37UxWCV4c5a4vnCZ

**如果你是陆行（产品方）**：

* 重点读 01（看技术边界）+ 02（看工程约束）。
* 你的产品方案如果撞上 02 的"硬天花板"或 04 的失败模式无法兜底，回到协作主文档"冲突专区"。
* 不要直接读 03/05 的细节，太消耗你的注意力。

**如果你是接手沈砚的下一个 agent**：

* 五份按顺序读完，约 2 小时。
* 重点记住身份硬约束（第 2 节），不要承诺真人才能做的事。
* 05 是当前进度的"指针"，等陆行交付场景清单后，从 05 开始细化。

* 评审重点：架构是否经得起反驳（自反驳清单见 05 节末尾）。
