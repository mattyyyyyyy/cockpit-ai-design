## V1 关键放弃条件（任一触发则整功能不上线）

| 功能点 | 放弃条件 |
|--------|----------|
| 节律识别（B-F3） | <14天行为数据，或准确率<60% |
| 场景模式识别（S-F3） | 识别准确率<70%，或用户反馈切换错误率>20% |
| 习惯主动服务（D-F3） | 用户接受率<30%，或触发误报率>20% |
| 品类聚合Agent | <3个品类MCP接入，或路由准确率<80% |
| 多层支付安全 | 声纹活体识别准确率<95%（P95），或支付超时>5s（P95） |
| 虚拟形象Live2D | 驾驶中CPU/GPU额外占用>10%（影响安全功能） |

## 开发优先级排序（Milestone）

**Milestone 1（基础可用，M+3）**
感知层全部 Must + NLU三级路由 + 车控Sub-bot + 基础TTS + 简单卡片HMI

**Milestone 2（智能可用，M+5）**
记忆系统（B-F1/B-F2/B-F6） + 主动服务（D-F1/D-F2/D-F4~D-F7） + 核心App接入（8个） + 多屏协同

**Milestone 3（完整V1，M+7）**
场景模式 + 个性化人格 + 安全合规全覆盖 + 弱网降级 + 数据闭环 + Live2D形象

**V1.5 评估项（发布后4-8周根据数据决定）**
节律识别 + 习惯主动服务 + 多人格 + 跨端延续 + MCP完整路由
