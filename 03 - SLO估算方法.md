**03 - SLO估算方法**

**SLO 估算方法**

**文档版本**: v0.1**创建时间**: 2026-05-10**作者**: 沈砚(后端架构师 + Agent 编排)**目标**: 给出座舱 AI 的 SLO(服务等级目标)估算方法,可用于容量规划、性能优化、故障定位

**1. 延迟 SLO**

**1.1 端到端延迟公式**

从用户唤醒到收到反馈的总延迟:

|  |
| --- |
| Plain Text T\_total = T\_wake + T\_asr + T\_nlu + T\_agent + T\_tool + T\_tts |

各环节延迟:

* T\_wake: 唤醒词检测延迟(端侧)
* T\_asr: 语音识别延迟(端侧或云端)
* T\_nlu: 意图识别 + 槽位填充延迟(端侧或云端)
* T\_agent: Agent 编排延迟(串行累加或并行取最大)
* T\_tool: 工具调用延迟(车控/导航/娱乐)
* T\_tts: 语音合成延迟(端侧或云端)

**1.2 场景级延迟估算**

**场景 1: 简单车控(开空调)**

|  |
| --- |
| Plain Text T\_total = 200(wake) + 300(asr端侧) + 100(nlu端侧) + 50(agent) + 50(CAN) = 700ms P50: 700ms / P90: 900ms / P99: 1.5s(冷启动) |

**场景 2: 导航(POI + 路线)**

|  |
| --- |
| Plain Text T\_total = 200 + 500 + 300 + 200 + 800 + 500 = 2.5s P50: 2.5s / P90: 4s / P99: 8s(网络抖动) |

**场景 3: 复杂意图(边开车边导航边播音乐)**

|  |
| --- |
| Plain Text T\_total = 200 + 500 + 300 + 300 + max(500, 2000, 800) + 500 = 3.8s 优化: 流式反馈,先反馈车控(1.5s),再反馈导航(3.8s) |

**1.3 网络延迟模型**

**4G RTT**: P50 50ms / P90 100ms / P99 300ms / P99.9 1s+**5G RTT**: P50 20ms / P90 40ms / P99 100ms

**云端推理**: 7B 200-500ms / 13B 500ms-1s / 70B 1-3s**端侧推理**: 1.8B 100-300ms / 3B 300-800ms

**1.4 SLO 目标建议**

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 场景 | P50 | P90 | P99 | 降级阈值 |
| 简单车控 | <800ms | <1.2s | <2s | 3s |
| 导航 | <2.5s | <4s | <6s | 8s |
| 复杂意图 | <3s | <5s | <8s | 10s |
| 多轮对话 | <5s | <8s | <12s | 15s |

**2. 成功率 SLO**

**2.1 分层成功率**

|  |
| --- |
| Plain Text 成功率\_总 = 成功率\_ASR × 成功率\_NLU × 成功率\_Agent × 成功率\_Tool |

**2.2 各层估算**

**ASR**: 安静普通话 98% / 车噪普通话 95% / 车噪方言 90% / 高速方言 85%**NLU**: 高频 98% / 中频 95% / 长尾 90% / 模糊意图 80%**Agent**: 单一 99% / 简单组合 95% / 复杂组合 90% / 动态调整 85%**Tool**: CAN 99.9% / POI 99% / 音乐 98% / 第三方 95%

**2.3 场景级估算**

* 简单车控: 0.95 × 0.98 × 0.99 × 0.999 = 92%
* 导航: 0.95 × 0.95 × 0.95 × 0.99 = 85%
* 复杂意图: 0.95 × 0.90 × 0.90 × 0.98 = 75%

**2.4 SLO 目标**

|  |  |  |  |
| --- | --- | --- | --- |
| 场景 | 目标 | 下限 | 告警 |
| 简单车控 | >95% | >90% | <85% |
| 导航 | >90% | >85% | <80% |
| 复杂意图 | >85% | >80% | <75% |
| 多轮对话 | >80% | >75% | <70% |

**3. 可用性 SLO**

**3.1 公式**

|  |
| --- |
| Plain Text 可用性\_总 = 可用性\_云端 × 可用性\_网络 + (1 - 可用性\_网络) × 可用性\_端侧 |

**3.2 各层**

* 云端: 大模型/ASR/TTS/Agent 99.9% / 导航 99.5%
* 网络: 主干道 99% / 地库 60% / 高速 95% / 隧道 10% / 偏远 50%
* 端侧: CAN 99.99% / 端模型 99.9% / 缓存 99.9%

**3.3 场景估算**

* 主干道: 0.999 × 0.99 + 0.01 × 0.999 = 98.9%
* 地库: 0.999 × 0.6 + 0.4 × 0.999 = 99.9%(端侧兜底拉高)
* 隧道: 0.999 × 0.1 + 0.9 × 0.999 = 99.9%(端侧主导)

**3.4 SLO 目标**

* 车控 >99.9% / 导航 >99% / 娱乐 >98% / 第三方 >95%

**5.2 99% 预算**

* 年: 87.6 小时 / 月: 7.2 小时

**5.3 监控**

|  |
| --- |
| Plain Text 已消耗 = Σ故障时长 消耗速率 = 已消耗 / 已过去时间 预计耗尽 = 剩余 / 消耗速率 |

告警: >50% 黄 / >80% 红 / 预计本月耗尽 → 停发布

**6. 监控指标**

**6.1 黄金指标**

* 延迟 p50/p90/p99
* 流量 qps / DAU
* 错误 error\_rate / timeout\_rate
* 饱和度 cpu/mem/queue

**6.2 业务**

* ASR/NLU/Agent/Tool 成功率
* 用户满意度、重试率、放弃率
* 单请求/单用户成本

**6.3 告警规则**

|  |  |  |  |
| --- | --- | --- | --- |
| 指标 | 阈值 | 级别 | SLA |
| P99 >5s | 持续 5min | P2 | 30min |
| 错误率 >5% | 持续 3min | P1 | 15min |
| 可用性 <99% | 持续 10min | P0 | 立刻 |
| 成功率 <85% | 持续 10min | P1 | 15min |
| 预算消耗 >80% | 立刻 | P1 | 1h |

**7. 性能优化方法**

**7.1 延迟优化**

* **并行化**: asyncio.gather,total = max 而非 sum
* **流式返回**: async for chunk in agent.stream() 边生成边返回
* **预测式执行**: 置信度 >0.9 直接执行,否则确认

**7.2 成功率优化**

* 低置信度多轮澄清
* 云端超时切端侧兜底
* 用户点踩触发重试不同策略

**7.3 成本优化**

* 端云分级(高频端侧/长尾云端)
* 模型分级(简单 7B/复杂 70B)
* 缓存命中绕开模型
