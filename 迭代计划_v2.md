# 座舱 AI 迭代计划 v2

> 2026-05-13 | 逻辑：用户痛点 → 具体做法 → 达成效果

---

## 1. AI 记忆能力：说过的不用再说

**痛点**
用户习惯用省略和代词说话，"去老地方""老样子"这类表达现在 AI 完全不懂。跨会话也没有记忆，下周上车还是不认识你。

**两层记忆分开做**

**短期：当次会话内的指代解析**
在 `query_rewrite_service` 里增加指代词识别。识别到"老地方/上次那个/老样子"时，查当次会话历史或用户最近5条记录，把模糊词替换成具体实体（"老地方"→"张江高科技园区"）再走 NLU。实现方式是在改写的 few-shot prompt 里补充这类示例，加一个历史查询步骤。

**长期：跨会话偏好持久化**
新建 MySQL 表 `user_preferences`，字段为 `user_id / category / key / value / confidence / updated_at`。来源有两个：用户明确说"记住"时写入 confidence=1.0；同一选择重复3次自动写入 confidence=0.85。每次请求前把用户偏好注入 LLM context，让 AI 带着用户背景理解意图。

**歧义消除（独立问题，和老地方不同）**
当匹配到多个候选时（通讯录里有两个张伟），不走指代解析，而是 TTS 播报"您联系人里有两个张伟，是公司的还是家里的？"等用户回答再执行。是选择题，不是查历史。

**达成效果**
"老样子""老地方"直接识别，不再说"对不起我没理解"。偏好记住后跨会话不重复问。

## 2. 响应有反馈：用户不会以为 AI 死了

**痛点**
用户说了一句话，等了3秒没动静，不知道 AI 在处理还是没听到，再说一遍，结果两条一起执行了。

**做法**
加渐进式感知反馈，不改实际处理速度，只改用户感知：
- 0–800ms：不做任何提示（快速请求不需要）
- 800ms 后：Live2D 切换"思考"状态 + 动效
- 2500ms 后：TTS 播"稍等一下"
- 3000ms 后：如果云端还没回来，切端侧简化版（车控 + 简单问答）

同时加幂等 Key：每次工具调用带唯一 ID，用户重复触发时相同 Key 不重复执行，不会重复下单。

**达成效果**
用户知道 AI 在处理，不会重复触发。就算网络慢，体验是"慢了一点"而不是"卡死了"。

## 3. 危险操作可以撤销

**痛点**
AI 执行了一个操作，用户反应过来想取消，但已经来不及了。比如导航开始了、订单下了、支付付了。

**做法**
所有有副作用的操作执行完之后，TTS 播一句"已完成，10秒内说取消可以撤销"，5秒时再提醒一次。用户说"取消"就调对应工具的 cancel 接口：
- 导航：停止导航
- 下单：取消订单
- 支付：发起退款申请

撤销窗口只靠语音，不弹屏幕卡片，不分散驾驶注意力。10秒超时后锁定，TTS 告知"已无法撤销"。

另外行驶中（车速 > 10km/h）500元以上的支付直接拒绝，让用户驻车后再操作。

**达成效果**
用户有后悔的机会。行驶中不会触发大额支付。

## 4. 工具出错时 AI 主动说出来

**痛点**
某个工具超时了，AI 现在的处理是打个 warning 日志，用户侧看到的是一直转圈，不知道多久能好、要不要重说一遍，只能干等或者反复尝试。

**做法**
把"打日志吞掉"改成"给用户一个明确出口"：
- 工具超时：TTS 播"星巴克点单现在有点问题，要我帮您记下来稍后提醒吗？"
- 云端慢：TTS 播"网络慢了一点，我先帮您处理车内的操作"
- 完全离线：TTS 播"现在网络不好，我只能控制车内设备，其他稍后再试"

按工具独立熔断：某个工具连续3次失败就暂时停用30秒，不影响其他工具。一个工具挂了，其他功能照常用。

**达成效果**
出问题时用户知道发生了什么，有明确的下一步选项，不会对着转圈不知所措。

## 5. AI 主动推送

**痛点**
AI 现在完全被动，用户不说话它就没反应。快没电了不提醒、连驾2小时不提醒、每周三接孩子它不知道。

**具体怎么做（分三步）**

**第一步：规则推送（先上，确定性高）**

车机每10秒通过 WebSocket 上报一个状态包 `{speed, soc, location, timestamp}`。后端收到后跑规则检查：

| 触发条件 | 播报内容 | 用户可说 |
|---------|---------|---------|
| SOC < 20% + 附近3km有充电桩 | "电量偏低，附近有充电桩，要导航过去吗？" | 好的 / 不用 |
| 连续行驶 > 2h + 附近有服务区 | "已连续行驶2小时，前方有服务区，需要休息一下吗？" | 需要 / 继续走 |
| 进入常用停车场附近 | "快到了，要帮您找停车位吗？" | 要 / 不用 |

技术上：在现有 WebSocket 连接里加一个 FastAPI background task，每次收到状态包时跑规则检查，命中就生成推送消息通过 WebSocket 推给车机，车机 TTS 播报。不需要新增通信通道。

**防打扰规则**：每次出行最多主动推2条；行驶中只播 TTS，不弹卡片；同类被用户拒绝2次后静默30天。

**第二步：偏好驱动推荐（P1记忆做完后）**

基于用户偏好表里已知的规律，在合适时机主动说：
- 已知用户常在工作日早上点咖啡 → 工作日早上出发时问"要帮您提前下单美式吗？"
- 已知每周三下午接孩子 → 周三下午出发时问"要导航去学校吗？"

**第三步：节律学习（V2）**

AI 自己从历史行为里学规律，不用手写规则，不用用户设置。

**达成效果**
用户不用主动说，AI 在合适时机提醒合适的事。规则阶段先让用户感受到"比较贴心"，偏好阶段让用户感受到"它了解我"。
